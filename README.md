## Main Key Learnings

## What Makes a Password Strong?
- Use **12–16 characters minimum**
- Combine **uppercase, lowercase, numbers, symbols**
- Avoid dictionary words
- Use **random phrases or passphrases**
- Don’t reuse passwords
- Enable **Multi-Factor Authentication (MFA)**

## Common Password Attacks
| Attack Type | Description | Prevention |
|--------------|--------------|-------------|
| Brute Force | Tries all possible combinations | Use long, random passwords |
| Dictionary Attack | Uses common words | Avoid real words |
| Phishing | Tricks user into revealing password | Verify all links/emails |
| Credential Stuffing | Reuses stolen passwords | Use unique passwords everywhere |

---

## Conclude
This experiment showed how password complexity directly affects strength scores.  
Simple passwords like `password123` scored very low, while longer ones with mixed characters like `Tru$tNo1But@iAgent2025!` achieved 100%.  
Hence, **length + randomness + variety = stronger security**.
